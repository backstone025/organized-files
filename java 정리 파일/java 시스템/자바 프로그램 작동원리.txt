- 확장자 : .java	클래스 파일 : .class
- 편집과 컴파일 및 실행과정
	[ 소스코드 작성 - .java 파일 생성 ] -> [ 컴파일러에서 바이트 코드로 변환(.class 파일 생성) ] -> [ 자바 가상 기계(JVM)에 의해 실행 ]

	# 가상 머신(기계) : 컴퓨터 안에 있는 컴퓨터(물리적으로 존재X)

	# 자바 가상 머신(JVM / Java Virtual Machine)  : 소스 코드의 최적화및 컴파일러의 백엔드 담당한다.
		* 자바 실행 환경(JRE / Java Runtime Environment)에 포함되어 있음
		* c/c++와는 달리 라이브러리 함수와 작성한 여러 파일을 하나로 뭉치지 않고.
		  실행하는 와중에 가져와 쓰기만 한다. 

			# 클래스 로딩 : 자바 클래스 라이브러리(자바 API)에서 필요한 매소드를 끌어다 쓰는 것	<javaprograming 27pg 참고>

	* 자바 가상 머신쓰면 뭐가 좋음? 
			-> 개발환경(리눅스, 맥, 윈도우 등)에 상관 없이 자바 가상머신 깔려있으면 실행할 수 있음(c/c++은 불가능함)
			-> 코드 알아서 정리해줌(갈배지 컬렉터)
